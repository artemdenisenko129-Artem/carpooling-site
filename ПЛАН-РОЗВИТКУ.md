# План розвитку ПопуткиUA

## Що зроблено (станом на 2026-05-30)

### Фундамент
- Next.js 15 App Router + React 19 + TypeScript + Tailwind
- MongoDB Atlas (carpooling > announcements)
- Google OAuth + JWT авторизація
- Vercel деплой (auto на git push)

### UI / UX
- Головна сторінка: список з фільтрами (роль, район, тип, спільнота)
- Dropdown підказки спільнот при пошуку
- Форма /new: BlaBlaCar-стиль, автокомпліт міст, MapPicker, waypoints, туди-назад
- Карта /map: Leaflet CDN, маркери водій(В)/пасажир(П), без кластеризації
  - Неактивні: сірий + жовте кільце (#FFE500)
  - Активні: кольорові + біле кільце
  - Highlight по ?id= при переході з картки
- Окрема сторінка /announcement/[id]: SEO, JSON-LD, лічильник переглядів
- Кнопка "← Назад" у хедері сторінки оголошення
- Кнопка "🗺 Показати на карті" в розгорнутій картці списку
- Кнопка "Деталі →" в bottom sheet карти (зелена рамка)
- Профіль /profile: телефон, спільнота, Telegram handle
- Логотип: єдиний LogoSVG (дорога + 2 маркери), використовується скрізь

### Бекенд
- POST /api/announcements: створення + Telegram нотифікація в канал
- Cron /api/cron/archive: щодня 02:00 UTC, деактивує only "once" з минулою датою
- Адмін /admin: статистика, пагінація, bulk-деактивація, сортування (дата/перегляди)
- Лічильник views: findOneAndUpdate $inc при кожному відкритті

### Аналітика
- GA4 (G-7R9FDQHSGJ) через NEXT_PUBLIC_GA_ID в layout.tsx
- Власний лічильник views в MongoDB

### Telegram
- Канал @poputky_ua: бот @Poputtky_bot — адмін з правом публікації
- Автопублікація при новому оголошенні (без посилання — щоб не було превʼю реклами)

## Що можна зробити далі (пріоритет)

1. Push-нотифікації: підписка на маршрут, сповіщення при новому оголошенні
2. Пошук по зоні на карті (намалювати полігон)
3. Рейтинг/відгуки після поїздки
4. Верифікація телефону через SMS
5. Telegram login (видалили через складність, можна повернути пізніше)

---

## Архітектурні принципи

- Сайт головний, бот допоміжний
- Soft delete замість hard delete (isActive: false)
- Регулярні оголошення — ніколи не архівуються автоматично
- Leaflet тільки через window.L (CDN), ніяких async imports
- Один компонент = один файл, shared типи в types/
