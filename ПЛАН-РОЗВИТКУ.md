# План розвитку ПопуткиUA

**Останнє оновлення:** 2026-05-31

---

## Що зроблено (станом на 2026-05-31)

### Фундамент
- Next.js 15 App Router + React 19 + TypeScript + Tailwind
- MongoDB Atlas (carpooling > announcements)
- Google OAuth + JWT авторизація
- Vercel деплой (auto на git push) + cron jobs

### UI / UX
- Sticky header: пошук (Лупа1+Лупа2) + фільтри (роль, тип, спільнота)
- Логотип LogoSVG зменшений до 27px (пропорційний назві)
- Кнопки × на полях пошуку та спільноті — помаранчева рамка, помітні
- Фільтри: кнопка "Всі ×" з помаранчевим border при активному фільтрі (сигнал скинути)
- Autocomplete dropdown: z-index:9999, завжди поверх карти та інших елементів
- Форма /new: BlaBlaCar-стиль, автокомпліт міст, MapPicker, waypoints, туди-назад
- Карта: краплини-маркери (водій=червоний, пасажир=синій, єдина форма для обох ролей)
- Offset маркерів з однаковими координатами (deduplicateCoords, ~150м зміщення)
- Кольорова схема єдина скрізь: водій=червоний, пасажир=синій
- Bottom sheet при кліці на маркер
- Highlight маршруту при переході /?view=map&id=
- Окрема сторінка /announcement/[id]: SEO, JSON-LD, views++
- BackButton з ?back= параметром (надійна навігація між пристроями)
- Профіль /profile: телефон, спільнота, Telegram, мої оголошення

### Бекенд
- GET/POST /api/announcements (список + створення + Telegram нотифікація в канал)
- PATCH /api/announcements/[id] (деактивація власного)
- Cron /api/cron/archive: щодня 02:00 UTC, деактивує "once" з минулою датою
- Адмін /admin: статистика, пагінація, bulk-деактивація, сортування

### Аналітика
- GA4 (G-7R9FDQHSGJ)
- Лічильник views (MongoDB $inc, тільки на сторінці /announcement/[id])

### SEO
- Метадані, sitemap.xml, robots.txt
- SEO-сторінки міст /city/[slug]
- JSON-LD на сторінці оголошення
- Canonical через SITE.domain

### Telegram / AI
- Канал @poputky_ua: автопублікація при новому оголошенні (без посилання)
- Бот @Poputtky_bot: Google Gemini парсить вільний текст → структуровані поля

## Архітектурні принципи

- Карта — вкладка FeedPage (`/?view=map&id=`), не окрема сторінка `/map`
- Soft delete: `isActive: false` (ніколи не видаляємо записи)
- Регулярні оголошення — ніколи не архівуються автоматично
- Leaflet тільки через `window.L` (CDN afterInteractive), ніяких async imports
- Один компонент = один файл, shared типи в `types/`
- views++ тільки в page component, не в generateMetadata
