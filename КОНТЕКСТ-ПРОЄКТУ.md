## Карта (LeafletMap.tsx) — актуальний стан

**Маркери:**
- Форма: краплина (SVG teardrop) для обох ролей
- Водій: червона рамка (#E24B4A), сіра середина (неактивний) / червона (активний)
- Пасажир: синя рамка (#378ADD), сіра середина (неактивний) / синя (активний)
- Без прапорців, без кластеризації

**Offset дублів:**
- Якщо два маркери мають однакові координати (±0.0001°) — зміщуються на ~150м
- `deduplicateCoords()` перед рендером маркерів

**Z-index:**
- Map wrapper: `position:relative; zIndex:1` — ізолює Leaflet panes від sticky header
- Sticky header: `z-index:50` — завжди поверх карти
- PlaceAutocomplete dropdown: `zIndex:9999` — поверх всього

**Bottom sheet (при кліці на маркер):**
- Показує роль, маршрут, дату/дні, час, місця, спільноту
- Кнопки: Telegram контакт + "Деталі →" (з ?back= для повернення)

## Критичні технічні правила

**Leaflet + React 19:**
- CDN `<Script strategy="afterInteractive">` в layout.tsx
- Тільки `window.L` синхронно — ніяких `import("leaflet")` або `next/dynamic`
- Leaflet CSS НЕ в `<head>` (блокує рендер +530мс)

**CIFS mount (Cowork/sandbox):**
- Файли > ~5kb можуть обрізатися при записі через Edit/Write
- Перевіряти null bytes: `python3 -c "data=open(f,'rb').read(); open(f,'wb').write(data.replace(b'\x00',b''))"`
- Відновлювати з git: `git show HEAD:path/to/file > /tmp/file && python3 -c "...copy..."`
- git index.lock/HEAD.lock блокуються — видаляти тільки з PowerShell: `Remove-Item .git\index.lock -Force`

**PowerShell git:**
```powershell
git add -A
git commit -m "..."
git push
```

**TypeScript на Vercel:**
- Vercel запускає tsc при білді — помилки типів ламають деплой
- Sort з mongodb: `import type { Sort } from "mongodb"`

**views лічильник:**
- Тільки в `app/announcement/[id]/page.tsx` через `incrementViews()`
- `generateMetadata` використовує `findOne` без `$inc` (щоб не подвоювати)
